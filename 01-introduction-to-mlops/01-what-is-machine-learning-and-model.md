# Fundamentals

### What Is Machine Learning?

Think of machine learning like teaching a small child.

You don’t give the child rules like:

“If red + round → apple”

Instead, you show examples:

“This is an apple 🍎”

“This is a banana 🍌”

“This is a mango 🥭”

After seeing many examples, the child automatically understands patterns.

👉 Same with computers.

### 💻 Normal Programming vs Machine Learning

❌ Normal Programming

You tell the computer:

If petal length < 2 → Setosa
If petal length > 5 → Virginica

👉 You manually write rules.

### ✅ Machine Learning

You give:

Data (flower measurements)
Correct answers (flower names)

👉 Computer figures out rules itself

You give the computer:


| Petal Length | Petal Width | Flower     |
| ------------ | ----------- | ---------- |
| 1.4          | 0.2         | Setosa     |
| 5.1          | 1.8         | Versicolor |
| 6.0          | 2.5         | Virginica  |


👉 After seeing many rows like this…

The computer starts thinking:

“Oh… small petals → Setosa”
“Big petals → Virginica”

### 🧠 What Is a Model (Super Simple)

A model = what the computer learned

Think of it like:

👉 A brain inside the computer
👉 That remembers patterns

### 🔁 How It Works (Step-by-step)

1. You give data (150 flowers)
2. Computer studies patterns
3. It creates a model
4. You give a new flower:

Petal length = 5.1
Petal width = 1.8

5. Model says: “Versicolor”

### ⚡ One-Line Understanding

👉 Machine Learning = Learning from examples instead of rules
👉 Model = The knowledge learned from those examples
